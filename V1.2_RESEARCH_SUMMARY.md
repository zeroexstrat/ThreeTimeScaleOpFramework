## What's New in v1.2

### 1. KDE Bandwidth Selection: Silverman vs. Sheather-Jones

**Research Finding:** Silverman's rule ($h = 1.06\sigma n^{-0.2}$) is simple but suboptimal for multimodal distributions. Sheather-Jones plug-in method is more accurate but slower ($O(n^2)$ vs $O(n)$).

**Section Updated:** 14.2 (Curve Fitting Algorithms — KDE)

**Implementation Guidance:**
- Use Silverman for quick predictions when speed is critical
- Use Sheather-Jones when peer group PPH shows multiple modes (by belt family or phase)
- Benchmark both on your 620-sort corpus before committing

**Sources:**
- [KDEpy Documentation - Bandwidth Selection](https://kdepy.readthedocs.io/en/latest/bandwidth.html)
- [Sheather & Jones (1991) Method - ResearchGate](https://www.researchgate.net/figure/The-Improved-Sheather-Jones-bandwidth-selection-rule-in-Algorithm-1)

### 2. Kalman Filter Cold-Start Initialization: Three Strategies

**Research Finding:** Cold-start (no prior history) is a real bottleneck. Literature shows three approaches with different tradeoffs.

**Section Updated:** 14.4 (Kalman Filter for Sequential Updating)

**Three Methods:**
1. **Simple (Current):** Use peer mean as $x_0$, large $P_0$ → rapid convergence but ~30–60 min transient
2. **Bayesian:** Jointly estimate $x_0$ as random variable → reduces transient but higher cost
3. **Gaussian Process Optimization:** Off-line tuning of $(P_0, Q, R)$ → 80% RMSE improvement but corpus-dependent

**Recommendation:** Start with Method 1 (simplicity), benchmark Methods 2–3 offline to justify complexity.

**Sources:**
- [Kalman Filter Initialization - IEEE Xplore](https://ieeexplore.ieee.org/document/5979684/)
- [Initialization Without Assumptions - ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0005109820303824)

### 3. DTW Acceleration via Sakoe-Chiba Band Constraint

**Research Finding:** DTW is $O(mn)$, expensive on 620-peer group. Sakoe-Chiba band constraint reduces to $O(m \cdot r)$ when $r \ll n$.

**Section Updated:** 14.2 (Curve Fitting Algorithms — DTW)

**Implementation Guidance:**
- Applicable when warping path is expected to stay close to main diagonal (valid for sorts)
- Set band radius $r \approx 0.1 \cdot n$ (e.g., $r=24$ for 240-min sort) to allow ±10% temporal flexibility
- Reduces computation from $O(620 \times 240)$ to $O(620 \times 24)$ — ~10x speedup

**Sources:**
- [DTW Tutorial - rtavenar.github.io](https://rtavenar.github.io/hdr/parts/01/dtw.html)
- [Sakoe-Chiba vs. Itakura - IEEE Conference](https://ieeexplore.ieee.org/document/8778300/)

### 4. MLR Feature Scaling: Categorical + Continuous Mix

**Research Finding:** Don't standardize across categorical & continuous features. Categorical variables have no meaningful std dev.

**Section Updated:** 14.3 (Multiple Linear Regression)

**Best Practices:**
- **Categorical (DOW):** Code as indicator/dummy variables; do not scale
- **Continuous (SS Share, IDS Share, Vol/Head):** Center (subtract mean) to reduce multicollinearity, especially when adding interaction terms
- **Interactions:** When adding terms like DOW × SS Share, centering the continuous feature prevents inflation of standard errors

**Future Work:** Add interaction terms (DOW × Vol/Head, Peak × SS Share) and cross-validate to test if they improve RMSE.

**Sources:**
- [Interactions with Continuous Predictors - CRAN](https://cran.r-project.org/web/packages/interactions/vignettes/interactions.html)
- [Categorical & Continuous Interactions - The Analysis Factor](https://www.theanalysisfactor.com/interactions-categorical-and-continuous-variables/)
- [When to Standardize in Regression - Statistics By Jim](https://statisticsbyjim.com/regression/standardize-variables-regression/)

### 5. CUSUM Control Limits: Manufacturing Standards Applied

**Research Finding:** CUSUM is standard in warehouse/logistics operations. Requires two parameters: reference value $K$ and decision interval $H$.

**Section Updated:** 6.4 (Session-Level Trend — CUSUM Perspective)

**Parameter Guidance:**
- **$K$ (Reference Value):** Set to half the shift you want to detect. Example: to detect 5-point accuracy drop (0.85 → 0.80), use $K = 0.025$.
- **$H$ (Decision Interval):** Set to $5\sigma$ (standard), where $\sigma = \sqrt{\theta_0(1-\theta_0)/n}$ is the standard error per batch. Example with $n=20$ questions:
  $$\sigma \approx 0.045, \quad H = 5 \times 0.045 = 0.225$$

**Implementation:** Track cumulative sum $S_n$ per sorter over sessions. Flag when $|S_n| > H$ as significant improvement/degradation.

**Sources:**
- [CUSUM Control Charts - SPC for Excel](https://www.spcforexcel.com/knowledge/variable-control-charts/keeping-process-target-cusum-charts/)
- [CUSUM in Logistics - DocShipper](https://docshipper.com/uncategorized/control-chart-definition-logistics/)
- [CUSUM Parameters - JMP Documentation](https://www.jmp.com/support/help/en/19.1/jmp/cusum-control-charts.shtml)
